Chained time management

Intro

deepjs use and abuse of Chaning Pattern. First for time management - as all deepjs chained and asynchrone management is based on Promises - but also for syntax ergonomy.

For clarity and modularity, deepjs comes with a set of particular chains API, dedicated to specific tasks.

  • promise
  • deep
  • rest
  • fs
  • sh
  • ssh
  • ...

Each specific chain is based on Promise one and so could be seen as a "super-promise" with an extended API.

But remember that they are all based on same mecanism and behaviours.

Identities

When you manipulate such chain, you could always change the chain identity (so its API) by calling identities method. Each chain come with it's own identity method.

deep.fs("/my/root")				// we start a "file system" chain (nodejs) 
.from("myjsonfile.json")		// read a json file
.done(function(success){		// manipulate loaded json
	succes.myProp = "hello world";
	//...
})
.rest("myService")		// we take the "restful chain" API
.post()					// post current success (so maniplated loaded json) to myService
.fail(function(error){
	// handle any error produced somewhere in the chain
});
