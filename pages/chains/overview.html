Chained time management

At the beginning was the Promise...

deepjs use and abuse of Chaning Pattern. Before all for time management - as all deepjs chained and asynchrone management is based on Promises - but also for syntax ergonomy.

and then the Chains appear.

deepjs comes with a set of particular chains API dedicated to specific tasks.

  • deep : collects all the deepjs specific API (layers & queries).
  • restful : act as mediator for restful stores.
  • fs : manage async fs calls (nodejs).
  • sh : manage shell sessions and commands.
  • ssh : manage SSH sessions and commands.
  • ...

Each specific chain is based on Promise and so could be seen as a "super-promise" with an extended API.

But remember that they are all based on same mecanism and behaviours.

Identities

When you manipulate such chain (including Promise), you could always change the chain's identity (so its API) by calling identities method. Each chain come with it's own identity method.

deep.fs("/my/root")				// we start a "file system" chain (nodejs) 
.from("myjsonfile.json")		// read a json file
.done(function(success){		// manipulate loaded json
	succes.myProp = "hello world";
	//...
})
.rest("myService")		// we take the "restful chain" API
.post()					// post current success (so maniplated loaded json) to myService
.fail(function(error){
	// handle any error produced somewhere in the chain
});
