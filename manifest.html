deepjs

Abstracts

> The core.

The deepjs idea is to propose new generic ways of thinking and running javascript code.

In 2 words, it could be defined as "space-time programming".

Space

(i.e. taking care of the structure of - nested - objects that shape your app)

  • to design objects like "layers" :

    Any js object (structured and deep as you want) is seen like what we called a "layer". You can merge layers to create more complexe objects helped by tools that manage possible collisions on datas (merging rules) or functions (composition rules).
    This vision of objects as layers tend to create more modular/reusable/maintainable objects or applications.

    - we could apply layers on any objects by the bottom (like a background) : use it for classic OOP (inheritance, compostion).

    - And we could apply layers by the up (like a foreground) : use it to add aspects on your objects to specialise, add capabilities, modify or whatever.

    Those easy manipulation of layers are also really great about metadata management (think about JSON) !

    i.e. to design freely whatever object (OO meaning), as structured (functions, datas, ...) and deep as you want, by merging complex objects (seen as layers) together (kind of Aspect Oriented Programmation with deepness management)
  • to manipulate those "layers" with query :

    We could query existant object structures before applying them some aspects at precise(s) place(s) depending on a more or less complex filter.

    - It helps you modelise without knowing the complete structure of your objects/applications. Which gives by example some great tools as "code sheets" that could be compared with "styles sheets".

    - And it helps to select objects you want to manipulate in run-time ( could be compared to JQuery but with your code and not the DOM), that help about losing coupling.
  • to use specific inheritance tools as "local internal inheritance" (an object could inherit locally from a brother, a oncle, or a mix of them, etc.) or by describing, at any structure deepness what are the objects/aspects "backgrounds".

Time

(i.e. taking care of the (async) time management between function's calls)

The core handle async promises "auto-magically" for you at almost any level...

  • It use and abuse of the Promise Pipelining pattern (async programmation with "promises") and provides a set of 'async chains' (objects with specific chainable API to manage File System, a shell session (or ssh), a db transaction, a call to a services, etc. or simply datas transformations...
    Thanks to Promises : it hide differences between sync and async programmation.
  • When merging two objects, deepjs also manages time sequence between merged functions/methods. It mimics the Promise Pipelining pattern and make function's composition a pleasure (sync or not) and provides nice way of errors handling.
  • In addition, we could play with local concurrent contexts (particular namespace) that are associated to each asynch call and define their own context.
    Thanks to this pattern (that come from Futures and Kris Zyp work) : it really makes concurrent programmation easier.

Object Capability Model

All this together (layer tools + query + inheritances + asynch/promises + concurrent context) plus another set of tools to manage "dependance injections" allow us to provide one of the most practical implementations of the OCM (Object Capability Model) paradigm for javascript, which become suddenly really usable and elegant.

OCM should be seen here as the capacity to produce dynamically objects (OO meaning - or even simple primitive value) with a structure that depends on environnement flags - user role(s), production/dev flags, particular modes, etc. - and as said before, the environnement itself could be concurrently managed.

All together

All those tools give "space-time programmation" paradigm (OCM is a part of it, and classical OO is a particular case of it), and has really positive and deep implications in security or achitecture of code.

A poetic image could be :

The code is seen as landscape, with valleys and mountains (aka. stacks of 'aspects'), that we flatten in a big unique structure, which began to vibrate locally following async runs, with the possibility to create on the fly, at any point of this space, some parallels worlds depending on local environnement.
